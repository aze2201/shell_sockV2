Terminal Sharing via x509 Certificates

A secure method to share local terminals with a cloud proxy

About

This project facilitates terminal sharing with a cloud proxy using x509 certificates. Unlike traditional SSH setups, managing individual device keys centrally in `.ssh/authorized_keys` is not necessary. It enables IoT device connections via a UNIX SOCK file, simplifying the process.

Key Features

  • No need for managing individual SSH keys
  • Connections established via UNIX SOCK file
  • Secure setup with x509 certificates
  • Proxy Linux machine handling signalling and media for `/bin/bash`

How It Works

All IoT devices connect to the Proxy Linux machine using two ports, one for signalling and another for `/bin/bash` media.

The x509 certificate setup ensures a secure connection, requiring certificates signed by the same CA utilized by the server.
